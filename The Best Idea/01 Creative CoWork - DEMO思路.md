# Creative CoWork - DEMO 思路

> 原始想法和架构设计

---

## 背景判断

创作 AGENT 在 25 年主要面向低端用户做提效，26 年希望针对高端用户，做高级创作者美誉度最高的工具。

### 三个核心观察

**1. 高级用户需要最强的能力**
- 目前 AGENT 的基础智力和生成能力都处于二流梯队
- 不同效果需要不同模型，超创有自己的独特心得

**2. 需要完整的创作解决方案**
- 内容创作不是只有生成这一个环节
- 从文生文到产物交付，有很多生成外的环节被忽略

**3. 高级用户有独特的使用方式**
- 产品经理定义的固定路径往往不够全面
- 大众路径不是针对超创设计的

### 对应的问题

1. 超创需要 SOTA 模型
2. 超创需要自定义的能力使用方式
3. 超创需要一站式完成创作路径

---

## 命题要点

1. **SOTA 能力的引入**（尽量合规）
2. **场景的自定义化**
3. **尽量高的效率**完成生成任务

---

## 核心架构

```
┌─────────────────────────────────────────────────────────┐
│                         前端                            │
│  ┌─────────┐      ┌─────────┐      ┌─────────────┐     │
│  │  文件夹  │      │  预览区  │      │ AGENT对话框 │     │
│  └─────────┘      └─────────┘      └─────────────┘     │
└─────────────────────────────────────────────────────────┘
                           │
               ┌───────────┴───────────┐
               │      AGENT CORE       │
               └───────────┬───────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
┌────┴────┐         ┌──────┴──────┐        ┌────┴────┐
│  Skills │         │   SANDBOX   │        │  GEN UI │
└─────────┘         └─────────────┘        └─────────┘
```

---

## 核心 Feature

1. **AGENT 实现**：并行体验，类似 Claude Code 多开
2. **前端页面**：初步实现，干掉硬性 bug
3. **Skills 自由加载**：任意 skills 的加载、调用、展示
4. **GENUI 生成**：基于资产、流程、工具动态生成界面

---

## 其他想法

1. 创作一定是 **human in the loop**
2. 工具面向超创，但普通用户也可以成长为超创
3. 产品形态参考：**Claude Code 的创作版**，或 **Kuse**
