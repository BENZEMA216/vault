# Creative CoWork - MVP 需求

> MVP 功能需求文档，持续更新

---

## MVP 目标

**验证假设**：上下文聚合 + AGENT + 可定制工具/界面，能让超创效率质变。

**成功标准**：3-5 个导演朋友愿意用它完成一个真实项目。

**里程碑**：0215 前完成可试用版本

---

## 产品形态

### 整体布局

![[StoryCode Studio - Canvas Mode.png]]

**布局说明**：两栏 + 悬浮对话框

- **左侧**：Explorer 文件树
- **中间/右侧**：预览区 / 画布（占据主要空间）
- **悬浮**：Story Agent 对话框（可拖拽、最小化、最大化）

### 核心链路

```
打开项目 → 看到文件树 → 点击文件预览 →
打开 AGENT 面板 → 添加 Context File →
对话指挥 → AGENT 调用 Tool → 返回结果 → 保存到项目
```

**GENUI 说明**：

本质上就是预览区能渲染 HTML。AGENT 生成 HTML 字符串，容器渲染它。

```
用户："帮我生成一个分镜表"
         │
         ▼
AGENT 读取项目文件，生成 HTML
         │
         ▼
预览区渲染 HTML（iframe 或 HTML 容器）
         │
         ▼
用户点击 HTML 中的按钮 → postMessage → 发消息给 AGENT
```

典型场景：
- 分镜表展示（表格 + 图片引用）
- 快捷操作按钮（生成下一镜、批量生成等）
- 自定义数据可视化

### 5. AGENT Core

AGENT 的后端逻辑，参考 [[Claude Code 架构分析]]。

#### 5.1 分层 Agent 架构

```
┌─────────────────────────────────────────────┐
│              Main Agent (Sonnet)            │
│    - 任务编排 & 决策                         │
│    - 任务状态管理                            │
│    - 工具调用决策                            │
└──────────────────┬──────────────────────────┘
                   │ 委派
     ┌─────────────┼─────────────┐
     ▼             ▼             ▼
┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│   Explore   │ │   Execute   │ │    Plan     │
│   (Haiku)   │ │  (Sonnet)   │ │  (Sonnet)   │
│   只读工具   │ │   全工具    │ │   只读工具   │
└─────────────┘ └─────────────┘ └─────────────┘
```

**关键约束**：最多只有一层 Sub-Agent（不能再嵌套），保持架构简单。
