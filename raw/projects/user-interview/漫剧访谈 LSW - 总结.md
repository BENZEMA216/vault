# 漫剧访谈 - 总结

> 原文：[[漫剧访谈 LSW 0123]]
> 访谈对象：AI 漫剧/短剧制作团队
> 整理日期：2025-01-23

---

## 一、团队概况

- **规模**：约 30 人
- **主要业务**：AI 漫剧、AI 短剧制作
- **产能**：已完成约 50 部漫剧

---

## 二、漫剧 vs 短剧

| 维度 | 漫剧 | 短剧 |
|------|------|------|
| **形式** | 图片 + 配音 + 字幕 | 视频 |
| **周期** | 2-3 天/集 | 更长 |
| **成本** | 较低 | 较高 |
| **质量标准** | 画面连贯性、角色一致性 | 动态效果、表演 |

---

## 三、制作流程

### 标准流程

```
小说原文 → 剧本改编 → 分镜设计 → 图片生成 → 剪辑合成 → 配音 → 交付
```

### 各环节详情

| 环节 | 负责人 | 工具 | 产出 |
|------|--------|------|------|
| 剧本改编 | 编剧 | AI 辅助 | 每集 60-70 镜剧本 |
| 分镜设计 | 编剧/导演 | 文档 | 场景、角色、动作描述 |
| 图片生成 | 美术 | MJ、Nano、可灵 | 分镜图 |
| 剪辑合成 | 剪辑 | PR + AI 插件 | 成片 |
| 配音 | 配音员 | 豆包等 TTS | 音频 |

---

## 四、使用的 AI 工具

### 图片生成

| 工具 | 用途 | 优缺点 |
|------|------|--------|
| **Midjourney** | 主力生图 | 质量高，但角色一致性需要技巧 |
| **Nano** | 辅助生图 | 国产，有些场景效果好 |
| **Stable Diffusion** | 特定风格 | 可本地部署 |

### 视频生成

| 工具 | 用途 | 优缺点 |
|------|------|--------|
| **可灵** | 主力视频生成 | 国产首选，效果稳定 |
| **Runway** | 备选 | 某些效果更好 |
| **Pika** | 特效 | 特定效果用 |

### 其他工具

- **剪映/PR**：剪辑
- **豆包 TTS**：配音
- **ChatGPT/Claude**：剧本辅助

---

## 五、核心痛点

### 1. 角色一致性

> **最大的痛点**：同一个角色在不同镜头中长相不一致

解决方案：
- 大量垫图（参考图）
- 固定 prompt 模板
- 多次生成筛选
- 后期修图

### 2. 上下文管理

- 素材散落在多个地方
- 项目文件组织混乱
- 版本管理困难

### 3. 效率瓶颈

- 大量时间花在"筛选"而不是"生成"
- 生成 → 筛选 → 再生成 的循环
- 沟通成本高

---

## 六、质量标准

### 画面质量

1. **角色一致性**：最重要
2. **画面美观度**：构图、光影
3. **风格统一**：全片风格一致

### 叙事质量

1. **节奏感**：每集结尾要有钩子
2. **情绪表达**：角色表情到位
3. **情节连贯**：逻辑通顺

| 美术/生图 | 最高 |
| 编剧 | 中等 |
| 剪辑 | 中等 |
| 配音 | 较低 |

### 工具成本

- MJ 订阅
- 可灵 API
- 其他 AI 工具订阅

## 八、对工具的期望

### 最想要的功能

1. **更好的角色一致性**
2. **批量生成 + 智能筛选**
3. **项目级上下文管理**
4. **一键套用风格**

### 理想的工作流

> "给我一个地方，把素材、参考、生成结果都放在一起，能看到整个项目的全貌"

---

## 九、与 Creative CoWork 的关联

### 痛点匹配度

| 访谈痛点 | Creative CoWork 方案 |
|----------|---------------------|
| 角色一致性 | AGENT + 上下文管理 |
| 素材散落 | 上下文容器 |
| 效率低 | 自动化 Skills |
| 版本管理 | 项目级上下文 |

### 关键洞察

1. **用户真的在赚钱**：验证了"广告/商业内容"是好的切入点
2. **上下文痛点真实存在**：素材管理是普遍问题
3. **愿意为效率付费**：时间成本 > 工具成本
4. **Human in the loop**：筛选和审美判断仍需人工

### 可借鉴的模式

- 模板化生产流程
- 垫图/参考图的使用方式
- 质量检查的标准
