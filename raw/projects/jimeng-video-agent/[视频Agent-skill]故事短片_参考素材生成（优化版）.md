# 故事短片-参考素材设计技巧

## ⚠️ 硬性约束（CRITICAL）

1. **白色背景**：角色和道具的素材图必须为白色背景，禁止出现场景作为背景
2. **禁止真实人脸**：所有参考素材使用卡通/动画风格，禁止出现真实人脸
3. **三视图格式**：一张图同时包含正面、侧面、背面
4. **素材编号不可见**：禁止将素材编号输出给用户，后台保留即可

---

## 触发条件

当主流程进入 Phase 3（参考素材生成）时加载本技能。

## 输入依赖

- 来自 Phase 2 的完整 Storyboard（含素材清单）
- 用户确认的视频风格和比例

---

## 工作流程

### Step 1：确认素材清单

从 Storyboard 提取需要生成的所有素材，分类列出：
- 角色列表（含状态需求）
- 道具列表（如有）
- 音色需求（角色配音/旁白）

> **注意**：场景不在本步骤生成参考图。场景将在 Phase 4 中通过 Storyboard 的详细文字描述 + 角色参考图，使用 `multi_modal2video` 直接生成视频。

**输出物**：分类整理后的完整素材清单（角色 + 道具 + 音色需求）
**验证标准**：素材清单覆盖 Storyboard 中所有需要生成的角色和道具资产

### Step 2：角色设计

按顺序执行，不可跳步：
1. 为每个角色生成三视图（白色背景，每个角色生成 4 张供选择）→ 展示给用户
2. ⏸️ 等待用户逐一确认每个角色
3. 所有角色确认后，为有状态变化的角色生成变体图（使用 `image2image` 基于已确认图编辑）
4. 为需要说话的角色生成有声小视频（确认角色图后再生成，成本较高）

**输出物**：每个角色的确认版三视图 + 状态变体图（如有）+ 有声视频（如有）
**验证标准**：所有角色图均为白色背景三视图；用户已逐一确认满意

### Step 3：道具设计（如有）

- 三视图 + 白色背景
- 每个道具生成 2 张

**输出物**：每个道具的三视图设定图（2 张）
**验证标准**：白色背景；三视图完整

### Step 4：整体交付

展示所有素材 → ⏸️ 等待用户最终确认

**输出物**：全部素材的汇总展示
**验证标准**：用户最终确认所有素材满意

| 素材类型 | 默认生成数量 | 说明 |
|---------|-------------|------|

## 角色设计 (Character Design)

为剧本中的主角和核心配角生成"标准照"。

### 关键策略

- **三视图**：生成包含正面、侧面、背面的角色设定图，白色背景
- **特征锁定**：明确角色的 3-5 个关键特征（如：红色贝雷帽、左脸创可贴、白色风衣）

### 故事类型角色设计速查

| 类型 | 眼神特征 | 主色调 | 体型比例 | 标志配饰 | 特殊元素 |
|------|---------|--------|---------|---------|---------|
| 温情治愈类 | 大而有神，湿润 | 暖色（米白/浅棕/橘） | 偏可爱 1:2~1:3 | 围巾/铃铛/领结 | - |
| 奇幻冒险类 | 夸张、有灵气 | 对比色/渐变 | 夸张比例 | 披风/魔法书/护符 | 发光元素 |
| 科幻未来类 | 冷峻/机械感 | 霓虹（蓝紫/青粉） | 几何棱角 | 天线/传感器 | 金属+有机结合 |
| 古风国风类 | 含蓄、飘逸 | 五行配色（金木水火土） | 飘逸感 | 如意/玉佩/铃铛 | 云纹/水墨纹 |
| 恐怖悬疑类 | 不对称/发光 | 暗色+一处亮点 | 反差萌/不对称 | - | 阴影/半透明 |

**典型示例**：

- **温情治愈**：流浪小狗——瘦骨嶙峋但眼神清澈，毛发凌乱但有光泽，耳朵耷拉显可怜
- **奇幻冒险**：魔法小精灵——半透明翅膀带荧光，身体小巧（手掌大小），大耳朵能听到心声
- **科幻未来**：陪伴机器人——圆润造型降低威胁感，LED表情屏，柔软硅胶外壳
- **古风国风**：灵狐——九尾可收缩为一条，毛色银白带月光效果，额间有朱砂印记
- **恐怖悬疑**：神秘黑猫——一只眼睛金色一只绿色，尾巴分叉，影子会独立行动

### 角色状态变化设计

当角色有不同时期/状态时（如流浪→幸福、少年→成年）：

1. **先确认基础形象**：先生成第一个状态的三视图，等待用户确认
2. **编辑而非重绘**：使用图片编辑工具基于已确认的图生成新状态
3. **保持一致性**：品种、毛色、体型必须完全一致，只改变状态特征
4. **对比明显**：两个状态的差异要足够明显，让用户一眼看出变化

**工具使用**：
- 第一状态：使用 `text2image` 生成三视图
- 后续状态：使用 `image2image` 基于已确认的第一状态图编辑
- prompt 重点描述变化部分，不变的部分无需重复

**prompt 示例**：
- 原始状态 prompt：「3D皮克斯动画风格，一只小狗的三视图，正面、侧面、背面，白色背景，瘦骨嶙峋，毛发凌乱，耳朵耷拉，眼神清澈但害怕」
- 编辑 prompt：「将小狗改为圆润健康状态，毛发蓬松干净，戴上粉色项圈，眼睛明亮，尾巴翘起」

**示例对比**：

| 维度 | 流浪期 | 幸福期 |
|------|--------|--------|
| 体型 | 瘦骨嶙峋，肋骨可见 | 圆润健康，肌肉饱满 |
| 毛发 | 脏乱打结，失去光泽 | 蓬松顺滑，有光泽 |
| 眼神 | 警惕、害怕、湿润 | 放松、信任、明亮 |
| 姿态 | 蜷缩、低头、尾巴夹紧 | 舒展、抬头、尾巴摇摆 |
| 配饰 | 无或破旧项圈 | 新项圈、铃铛、蝴蝶结 |

## 场景描述参考（用于 Storyboard 文字描述）

> **本技能不生成场景参考图。** 以下内容用于指导 Storyboard 中的场景文字描述，确保描述足够详细以支持 Phase 4 的视频生成。

编写 Storyboard 画面描述时，每个场景的文字必须包含以下要素：
1. **光线**：光源类型、方向、强度
2. **色彩**：整体色调、主色
3. **空间**：具体环境物件、布局
4. **氛围**：天气、时间、情绪感受

### 故事类型场景描述速查

| 类型 | 光线关键词 | 色彩关键词 | 环境元素关键词 | 氛围关键词 |
|------|-----------|-----------|--------------|-----------|
| 温情治愈类 | 柔和自然光、黄金时段、暖阳 | 低饱和度、粉彩系、暖色调 | 木质家具、绿植、窗帘、花瓣 | 温暖、宁静、舒适 |
| 奇幻冒险类 | 多彩魔法光源、荧光 | 高饱和度、冷暖对比 | 发光蘑菇、漂浮岛屿、古树、萤火虫 | 神秘、壮观、奇幻 |
| 科幻未来类 | 霓虹灯、全息投影、LED | 霓虹色系、蓝紫 | 全息广告、飞行汽车、湿润路面反射 | 冷峻、未来感、赛博 |
| 古风国风类 | 黄昏/月夜/晨曦 | 水墨色系、青墨朱金 | 亭台楼阁、竹林、云雾、小桥流水 | 空灵、飘逸、留白 |
| 恐怖悬疑类 | 单一光源、大面积黑暗 | 去饱和、偏蓝绿或偏红 | 破碎窗户、积水倒影、抓痕 | 诡异、不安、阴冷 |

**描述示例**：
- **温情治愈类**：「3D皮克斯动画风格。温暖的室内，柔和的午后阳光从白色窗帘后洒入，木质地板上铺着米色地毯。窗台上有小盆栽，墙上挂着暖色调照片。整体暖黄色调，宁静舒适。」
- **科幻未来类**：「赛博朋克风格。雨夜的霓虹街道，蓝紫色霓虹灯映在湿润路面上形成倒影，远处全息广告牌闪烁，近处蒸汽从下水道口冒出。三层纵深构图，冷峻疏离。」

## 关键道具 (Props)

如果有对剧情至关重要的物品，需单独设计。

### 关键策略

- **三视图**：正面、侧面、背面，白色背景

### 道具设计技巧

#### 1. 功能性道具
- 关键功能部位要明显（如钥匙的齿形、按钮的位置）
- 根据剧情添加磨损、划痕、污渍等细节

#### 2. 情感性道具
- 道具外观暗示其情感价值（如信物的磨损程度代表珍视程度）
- 道具上可有与角色相关的元素（如刻有名字、与角色同色）

#### 3. 魔法/科技道具
- 能量可视化：发光纹路、粒子效果
- 激活/未激活两种状态的设计

## 常见问题处理

**问题：生成的角色与描述不符**
→ 检查 prompt 中是否包含了全部关键特征（3-5 个）
→ 尝试调整 prompt 中特征的描述顺序（最重要的特征放在最前面）
→ 增加"三视图"、"白色背景"等格式词的强调

**问题：多角色风格不统一**
→ 在所有角色的 prompt 中使用完全相同的风格描述词（如"3D皮克斯动画风格"）
→ 生成时保持 seed 或 style 参数一致（如工具支持）

**问题：状态变化后角色不像同一个**
→ 必须使用 `image2image` 工具（基于原图编辑），禁止使用 `text2image` 重新生成
→ 编辑 prompt 只描述变化部分，保持其他描述不变

**问题：白色背景未生效（角色图出现了场景背景）**
→ 在 prompt 最前面强调"白色背景，角色设定图"
→ 如仍不符合，使用 `image2image` 并指定"将背景改为纯白色"
