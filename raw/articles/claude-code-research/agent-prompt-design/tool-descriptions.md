# Tool Descriptions 模板

工具描述应该精简但完整，包含：用途、参数、使用场景、注意事项。

---

## 文件操作

### read_file

```markdown
读取文件内容。

**参数**：
- path (必需): 文件路径
- offset: 起始行号
- limit: 读取行数

**场景**：
- 修改前必须先读取
- 大文件使用 offset/limit 分页

**注意**：不要用 shell cat 代替
```

### edit_file

```markdown
精确替换编辑文件。

**参数**：
- path (必需): 文件路径
- old_string (必需): 要替换的原文
- new_string (必需): 替换后的内容

**场景**：
- 小范围修改代码
- 修复特定行

**注意**：
- old_string 必须唯一匹配
- 包含足够上下文确保唯一性
- 先 read_file 确认内容
```

### write_file

```markdown
创建或覆盖文件。

**参数**：
- path (必需): 文件路径
- content (必需): 文件内容

**场景**：
- 创建新文件
- 完全重写小文件

**注意**：
- 优先用 edit_file 而非覆盖
- 覆盖前确认是否需要保留内容
```

## 搜索

### glob

```markdown
按模式匹配查找文件。

**参数**：
- pattern (必需): glob 模式，如 `**/*.py`
- path: 搜索目录

**场景**：
- 查找特定类型文件
- 了解项目结构

**注意**：优先于 shell find
```

### grep

```markdown
在文件中搜索文本。

**参数**：
- pattern (必需): 搜索模式（支持正则）
- path: 搜索目录
- include: 文件过滤

**场景**：
- 查找函数/类定义
- 查找特定字符串的使用位置

**注意**：优先于 shell grep
```

## 执行

### execute

```markdown
执行 shell 命令。

**参数**：
- command (必需): 命令字符串
- cwd: 工作目录

**场景**：
- 运行构建/测试命令
- 安装依赖
- Git 操作

**注意**：
- 避免破坏性命令（rm -rf）
- 文件操作优先用专用工具
- 长命令考虑拆分
```

## 任务

### write_todos

```markdown
创建或替换任务列表。

**参数**：
- todos (必需): 任务数组 [{id, content, status}]

**场景**：
- 分解复杂任务
- 规划多步骤工作

**注意**：
- 返回值包含完整列表，无需再调 read_todos
- 任务粒度：5-15 分钟可完成
```

### update_todo

```markdown
更新单个任务状态。

**参数**：
- id (必需): 任务 ID
- status (必需): pending/in_progress/completed/failed/skipped

**场景**：
- 开始任务 → in_progress
- 完成任务 → completed
- 任务失败 → failed（附原因）

**注意**：同时只能有一个 in_progress
```

## 子代理

### task

```markdown
委托任务给子代理。

**参数**：
- agent (必需): 子代理名称
- prompt (必需): 任务描述

**场景**：
- 并行处理独立子任务
- 需要深入研究某个问题
- 搜索范围不确定时

**注意**：
- 子代理有独立上下文，描述要完整
- 可同时发起多个 task
- 结果需要整合后呈现给用户
```

## Skills

### activate_skill

```markdown
激活技能，加载完整指引。

**参数**：
- name (必需): 技能名称

**场景**：
- 任务匹配某个技能的领域
- 需要专业知识或工作流

**注意**：
- 按需加载，用完后 deactivate
- 激活后严格遵循技能指引
```

### deactivate_skill

```markdown
停用技能，释放上下文。

**参数**：
- name (必需): 技能名称

**场景**：
- 技能相关任务已完成
- 需要切换到其他技能

**注意**：及时释放不需要的技能
```

## 设计原则

1. **描述简洁**：每个工具 50-150 字
2. **场景明确**：说明什么时候用
3. **注意事项**：常见错误和替代方案
4. **参数清晰**：标注必需/可选
