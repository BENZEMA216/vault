# 0804 - 【学习】看下 oppo 的 AGNET 工作

> 最让我好奇的是，他们是怎么实现对用户数据的利用和 AGENT 增强的
> 地址：https://github.com/OPPO-PersonalAI/Agent-KB

### 问题定义
- 问题定义：
  - （1）任务特定经验隔离 -- 智能体难以在不同任务类型间转移知识，在遇到新领域时，无法利用相关任务类别中的成功策略，只能从零开始。
  - （2）单一层级的检索粒度 -- 现有系统采用统一的检索机制，无法区分问题解决的不同阶段。初始规划需要基于问题特征检索到的高层级工作流指导，而执行中期的优化则需要基于当前执行轨迹检索到的细粒度推理修正。
  - （3）静态经验重放 -- 系统以原始形式存储和复用经验，缺乏适当的抽象，这使得经验无法有效适应新情境，而在新情境中，抽象原则往往比原始执行细节更能有效迁移。
### 怎么做
- 举了一个在 PDB 任务里，这个方法的案例，其实就是在原本的 tool use 的基础上，加了一个来自老师的经验，看图的右上角，然后任务成功率就增加了

- 学生智能体从库里面获取整体的工作流程模式，得到策略知道
- 老师智能体找出具体执行上的模式，帮助优化细节
我们的基本假设是，从较简单任务中抽象出的经验，通过捕捉高层级的规划模式而非特定框架的实现细节，能够被泛化并应用于新的、更复杂的问题
- 两个智能体都使用互补的 "推理 - 检索 - 优化"（RRR）循环来解决复杂任务。在这一架构中，执行智能体负责实际的任务解决，学生智能体检索工作流层面的经验以提供初始的策略指导，教师智能体则分析执行轨迹以识别错误，并检索步骤层面的经验进行优化。教师通过检测推理缺陷，并基于 AGENT KB 中的相关经验提供纠正性反馈，对学生进行监督。值得注意的是，这两个辅助智能体都专注于经验检索和自适应优化，以提升执行实际问题解决任务的执行智能体的表现。

- 推理问题的目标
- 检索工作流层面的经验
- 然后基于经验做 refine

- 总结推理步骤
- 识别错误、类型和原因
- 从经验里召回内容
- 给学生反馈
### 效果怎么样

代码和检索任务，在任意模型上都提升了（简直是废话）

- 若移除学生智能体或教师智能体中的任何一个，系统性能都会降至 59.39%
- 这个实验很充分地验证了两个具有差异性上下文的 AGENT 互相协作的有效性
