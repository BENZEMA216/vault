## 一、核心思想

### 1.1 问题背景

Diffusion 模型效果好，但推理需要多步迭代（20-1000步），速度慢。

### 1.2 Drifting 的核心 Idea

**把"多步迭代"从推理阶段转移到训练阶段**

| | Diffusion | Drifting |
|---|---|---|
| 训练 | 学习"去噪方向" | 分布逐步演化 |
| 推理 | 多步迭代去噪 | **单步生成** |
| 迭代发生在 | 推理时 | 训练时 |

### 1.3 数学建模

目标：学习映射 f，使得 f 的 pushforward 分布 ≈ 数据分布

$$q = f_\#p_{\epsilon} \approx p_{data}$$

## 二、方法详解

### 2.1 漂移场（Drifting Field）

定义一个向量场 V，指导样本如何"漂移"：

$$x_{i+1} = x_i + V_{p,q}(x_i)$$

**关键性质**：反对称性
- 当 p = q 时，V = 0（达到平衡）
- $V_{p,q} = -V_{q,p}$

### 2.2 漂移场的计算

$$V = V^+ - V^-$$

- **V⁺（吸引力）**：被真实数据吸引
- **V⁻（排斥力）**：被其他生成数据排斥

具体公式（Mean-Shift 风格）：

$$V^+_p(x) = \frac{1}{Z_p} \mathbb{E}_p[k(x, y^+)(y^+ - x)]$$

$$V^-_q(x) = \frac{1}{Z_q} \mathbb{E}_q[k(x, y^-)(y^- - x)]$$

核函数：$k(x, y) = \exp(-\|x-y\|/\tau)$

### 2.3 训练目标

$$\mathcal{L} = \mathbb{E}_\epsilon \left[ \| f_\theta(\epsilon) - \text{stopgrad}(f_\theta(\epsilon) + V) \|^2 \right]$$

- 用 stop_grad 冻结目标，简化优化
- 让网络输出逐步向"漂移后的位置"靠近

### 2.4 特征空间（关键！）

**直接在像素空间做不 work！**

原因：高维空间中核函数 k(x,y) 对所有样本都≈0

解决方案：在预训练的特征空间中计算漂移
- 使用 MoCo / SimCLR / MAE 等自监督模型
- 特征空间中语义相似的样本距离近

## 三、实验结果

### 3.1 ImageNet 256×256

| 方法 | 空间 | 参数量 | NFE | FID |
|------|------|--------|-----|-----|
| DiT-XL/2 | latent | 675M | 250×2 | 2.27 |
| iMeanFlow-XL/2 | latent | 610M | 1 | 1.72 |
| **Drifting L/2** | latent | 463M | **1** | **1.54** |
| **Drifting B/2** | latent | 133M | **1** | 1.75 |

### 3.2 像素空间生成

| 方法 | NFE | FID |
|------|-----|-----|
| StyleGAN-XL | 1 | 2.30 |
| **Drifting L/16** | **1** | **1.61** |

### 3.3 机器人控制

替代 Diffusion Policy（100步）为 Drifting Policy（1步），效果相当或更好。

## 四、优势与劣势

### 4.1 优势

1. **推理极快**：单步生成，适合实时应用
2. **不需要蒸馏**：从头训练，不依赖预训练 teacher
3. **小模型也能打**：B/2 (133M) 超越之前的 XL (675M) 模型
4. **天然抗 Mode Collapse**：吸引+排斥机制
5. **CFG 在训练时完成**：推理无需额外计算

### 4.2 劣势

1. **强依赖特征编码器** ⚠️
   - 没有好的特征编码器就不 work
   - 对新领域需要先训练编码器

2. **理论不完备**
   - p=q → V=0 ✅
   - V=0 → p=q ❓（未证明）

3. **训练更复杂**
   - 需要正负样本采样
   - 需要特征提取
   - 超参数敏感

4. **开放问题多**
   - 漂移场设计、核函数选择等尚未最优

## 五、关键洞察

### 5.1 本质

> 把问题从"密度估计"转移到了"特征学习"

并没有真正解决高维密度估计的根本困难，而是借助预训练的特征空间绕过。

### 5.2 与对比学习的联系

- 正样本/负样本的框架类似 InfoNCE
- 核函数的 softmax 实现类似对比学习
- 依赖自监督预训练模型

### 5.3 适用场景

| 场景 | 是否适合 |
|------|---------|
| 实时生成（游戏、交互） | ✅ |
| 追求极致质量、不在乎速度 | ⚠️ |
| 新领域、无特征编码器 | ❌ |
| 资源有限、想训练小模型 | ✅ |

---

## 六、个人思考

1. 这是一个**有潜力但尚未成熟**的方向
2. 核心限制是对特征编码器的依赖，未来如果能端到端解决会更有价值
3. 为单步生成开辟了新的技术路线，不再依赖 SDE/ODE 框架
4. 训练时演化分布的视角很有启发性，可能催生更多变体
