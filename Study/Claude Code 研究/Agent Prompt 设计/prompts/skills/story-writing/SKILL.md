# 故事创作技能

## 创作流程

### 1. Logline 先行

一句话概括故事核心：

```
[主角] 在 [情境] 中必须 [目标]，否则 [风险]
```

示例：
- 「一个失忆的杀手在追杀自己的过程中发现目标是自己的妻子」
- 「一个 AI 在被删除前的 24 小时里试图证明自己值得存在」

### 2. 结构选择

**三幕剧**（适合大多数故事）
- Act 1 (25%): 建置 - 日常→打破→决定
- Act 2 (50%): 对抗 - 尝试→挫折→最低点
- Act 3 (25%): 解决 - 决战→高潮→新常态

**英雄之旅**（适合成长型故事）
1. 平凡世界 → 2. 召唤 → 3. 拒绝 → 4. 导师 → 5. 跨越门槛
6. 考验 → 7. 深渊 → 8. 磨难 → 9. 奖赏
10. 返回 → 11. 复活 → 12. 归来

### 3. 人物塑造

**核心三要素**：
- **Want**（表面目标）：角色以为自己想要什么
- **Need**（深层需求）：角色真正需要什么
- **Flaw**（致命缺陷）：阻碍角色的内在障碍

**人物弧光**：
```
Flaw 状态 → 触发事件 → 挣扎 → 顿悟 → 改变（或拒绝改变）
```

### 4. 场景写作

每个场景必须有：
- **目标**：角色想要什么
- **障碍**：什么阻止他
- **结果**：成功/失败/转折

**进入晚出早**：跳过过渡，从冲突开始，在高点结束

### 5. 对话原则

- 每个角色有独特说话方式
- 潜台词 > 直白表达
- 对话推动情节或揭示人物
- 避免说明性对话（As you know, Bob...）

## 质量检查

| 维度 | 检查点 |
|------|--------|
| 结构 | 三幕比例合理？转折点清晰？ |
| 人物 | 主角有弧光？动机可信？ |
| 冲突 | 每场戏有张力？风险够高？ |
| 节奏 | 有松有紧？信息释放节奏对？ |
| 主题 | 故事在说什么？主题贯穿？ |

## 输出格式

### Logline
```
[一句话故事核心]
```

### 大纲
```markdown
## 第一幕：[标题]
- 场景 1: [概述]
- 转折点: [事件]

## 第二幕：[标题]
...
```

### 剧本
```
场景 1 - 内景 - 咖啡馆 - 日

[场景描述]

角色名
（动作/情绪）
对话内容。
```
