# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目概述

这是一个 Obsidian 知识库，包含 **Creative CoWork** 产品的设计文档和学习笔记。

Creative CoWork 是面向专业创作者（"超创"）的 AI 创作工作台，定位为"Claude Code 的创作版"。

## 目录结构

```
WORK/
├── The Best Idea/     # 核心产品文档
│   ├── 01-04 *.md     # Creative CoWork 设计文档
│   └── resources/     # 相关资源
├── Study/             # 学习笔记和论文
├── User Interview/    # 用户访谈记录
└── DEMO INTRO/        # Demo 相关材料
```

## 核心概念

- **超创**：能用 AIGC 赚钱的专业创作者（目标用户）
- **上下文容器**：聚合项目所有上下文（素材、生成结果、参考、对话历史）
- **AGENT**：自然语言交互，具备 L3-L4 智能（上下文利用 + 多步编排）
- **Studio**：场景化工作空间模板（工具 + 界面 + 配置）
- **Skills**：用户可自由加载的模块化能力
- **GENUI**：基于 skills/上下文动态生成的界面
- **SANDBOX**：代码执行环境，用于编排和执行任务

## 编辑规范

- 所有文档使用中文
- 使用 Obsidian 风格的 Markdown（支持 `[[链接]]` 语法）
- 更新文档时保持现有结构和编号
- 添加新内容时注意与相关章节的交叉引用
- Study 文件夹用于存放论文笔记和学习材料
- **下载的论文 PDF 统一放在 Study/论文 文件夹**
- **User Interview 文件夹中的访谈原文禁止修改**，只能新建文件做总结或分析

## 访谈总结规范

### 文件命名
- 总结文件命名格式：`{原文件名} - 总结.md`
- 示例：`漫剧访谈 三界动画 - 总结.md`

### 文件头部（元信息）
```markdown
# 漫剧访谈 XXX - 总结

> 原文：[[原文件名]]
> 访谈对象：公司名 - 受访人（职位）
> 访谈时长：X小时X分钟
> 整理日期：YYYY-MM-DD
```

### 必须包含的章节

| 章节 | 内容 |
|------|------|
| 一、公司概况 | 基本信息、业务类型、商业模式 |
| 二、团队组织 | 角色分工、人数、协作方式 |
| 三、产品类型与成本 | 成本分级、产能基准 |
| 四、制作流程 | 完整流程图 + 各环节详情 |
| 五、使用的工具 | 工具矩阵表（环节-工具-原因） |
| 六、核心痛点 | 分类列出痛点和表现 |
| 七、质量评判标准 | 底层要求 + 进阶要求 |
| 八、对工具的期望 | 理想状态描述 |
| 九、与 Creative CoWork 的关联 | 痛点匹配表 + 关键洞察 |
| 十、值得注意的细节 | 访谈中的有趣发现 |
| 十一、金句摘录 | 有价值的原话引用 |
| 十二、待深挖问题 | 后续可追问的点 |

### 格式要求
- 使用表格呈现对比信息（角色分工、成本分级、工具矩阵、痛点列表）
- 使用 `>` 引用重要金句
- 使用 ``` 代码块展示流程图
- 章节之间用 `---` 分隔
- 文件末尾标注整理日期
