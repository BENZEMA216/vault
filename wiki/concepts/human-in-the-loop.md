# 人机协作（Human in the Loop）

> Agent 系统中人类参与决策的机制，在创意领域尤为关键：人定方向、AI 出活。

## 核心要点

- 完全自动化不是目标，人类把控方向和关键决策、AI 执行技术性工作才是理想模式
- 创意领域的 HITL：导演把控故事板和美学决策，AI 负责素材生成和技术实现
- 用户访谈一致确认：即使 AI 能力再强，创作者仍需要掌控感
- 实现方式：阶段性确认（视频 Agent 每阶段确认）、浮动对话框交互、选择题而非开放题
- 战略定位：HITL 是产品差异化的关键，纯自动化产品无法满足专业创作者需求

## 详细说明

Human in the Loop 在 Agent 系统中不是一个权宜之计，而是一个核心设计原则。产品深化文档的 Q6 战略问题明确提出：Agent 的目标不是取代人类创作者，而是成为创作者的"超级工具"。这个定位直接影响了系统架构——Agent 需要在关键节点暂停、等待人类决策、然后继续执行。

在所有用户访谈中，无论团队规模大小，创作者都表达了相同的需求：他们希望 AI 处理重复性、技术性的工作（如生成大量素材变体、自动调色、批量渲染），但故事方向、角色设定、美学风格等核心创意决策必须由人类把控。三界动画的导演明确表示，AI 生成的故事板需要人工审核和调整后才能进入制作流程。艺柏 AILab 的实践也证明，剪辑节奏和叙事结构是人类创作者不愿让渡的领域。

在具体实现上，视频 Agent 的主 SP 设计了每阶段确认机制：需求分析完成后等待用户确认方向，故事板完成后等待用户审核分镜，素材生成后等待用户选择满意的版本。MVP 需求文档中的浮动对话框交互设计也体现了这一原则——Agent 通过轻量级 UI 而非全屏接管的方式请求用户决策，最大限度减少打断感。
